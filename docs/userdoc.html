This section provides information for Asmeta users and people interested in the Asmeta project.

AsmetaL language

For information about the language we have derived from the metamodel, examples and a quick guide how to write specifications in this language, see the AsmetaL documentation.

  • A quick guide of the concrete notation
  • A more complete guide about the language
  • The EBNF grammar for the AsmM textual notation. It has been developed by means of the Java Compiler Compiler (JavaCC)
  • The railroad grammar generated by https://www.bottlecaps.de/convert/

AsmetaL examples

  • Several examples of ASM specifications (mono agent and multi agents) can be found under github here
  • New models can be found also here

AsmetaL Standard Libraries

  • The AsmM Standard Library A library of predefined ASM domains and functions: StandardLibrary.asm (you need this)
  • LTL and CTL Standard Libraries: to be used together with the model checker to introduce TL properties: LTLLibrary.asm and [CTLLibrary.asm] (https://github.com/asmeta/asmeta/blob/master/asm_examples/STDL/CTLLibrary.asm)
  • Time libraries: to use time in Asmeta specs: TimeLibrary.asm and the simplified version TimeLibrarySimple

Asmeta Metamodel

Our first idea have been laid down in a paper presented at ASM'04.

A complete definition of the AsmM metamodel can be found in:

A. Gargantini, E. Riccobene, and P. Scandurra. Metamodelling a Formal Method: Applying MDE to Abstract State Machines. Technical Report 97, DTI Dept., University of Milan, November 2006. [link]

Here you find:

  • The AsmM Abstract Syntax (the metamodel)

    • The ecore file for the AsmM
    • (not yet in the metamodel, with some errors) The OCL constraints

From the AsmM we have derived the XMI format and the Java EMF libraries.
